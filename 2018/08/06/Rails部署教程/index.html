Rails部署教程

Posted on 2018-08-06 | In 学习笔记

Rails部署教程

0 基本工具

一台Centos的云服务器。最终的部署成功的环境：Ruby-2.1.5 nginx-1.6.2 mysql-5.7.22

1 添加deploy

adduser deploy 增加deploy用户

passwd deploy 修改deploy用户的密码

在root账户下 添加 /etc/sudoers.d/deploy deploy ALL=(ALL:ALL) ALL

2 deploy免密码登入

mkdir ~/.ssh

touch ~/.ssh/authorized_keys

回到本机，执行 cat ~/.ssh/id_rsa.pub，把公钥印出来

chmod 700 ~/.ssh chmod 644 ~/.ssh/authorized_keys

3 设置服务器闲置自动退出的时间

方法1

vi /etc/ssh/sshd_config

ClientAliveCountMax 60

service sshd reload

LoginGraceTime 20m

方法2

vi /root/.ssh/config

ServerAliveInterval 60

方法3

vi /etc/profile

TMOUT=1800

4 更新和安装Linux套件

sudo yum update (可以通过强制关掉yum进程：#rm -f /var/run/yum.pid)

sudo yum upgrade

sudo yum install -y build-essential git-core bison openssl libreadline6-dev curl zlib1g zlib1g-dev libssl-dev libyaml-dev libsqlite3-0 libsqlite3-dev sqlite3 autoconf libc6-dev libpcre3-dev libcurl4-nss-dev libxml2-dev libxslt-dev Imagemagick nodejs libffi-dev

sudo yum install ImageMagick-devel ImageMagick

5 安装rvm的依赖包

sudo yum install -y gcc-c++ patch readline readline-devel zlib zlib-devel libffi-devel openssl-devel make bzip2 autoconf automake libtool bison iconv-devel libyaml-devel curl-devel sqlite-devel

6 安装rvm

方法1

curl -sSL https://rvm.io/mpapis.asc | gpg –import -

curl -L get.rvm.io | bash -s stable

方法2

gpg –keyserver hkp://keys.gnupg.net –recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3

\curl -sSL https://get.rvm.io | bash -s stable

方法3

curl -L https://raw.githubusercontent.com/wayneeseguin/rvm/master/binscripts/rvm-installer | bash -s stable

source /home/deploy/.rvm/scripts/rvm

source ~/.profile

rvm install 2.1.5

rvm use 2.1.5 –default

gem sources –add https://gems.ruby-china.com –remove https://rubygems.org

gem install bundler

gem install bundle

查看gem是否变多的方法：du -sh /opt/apps/projects/sutong/shared/bundle/ruby/2.1.0/cache

7 安装数据库

wget http://repo.mysql.com/mysql57-community-release-el7.rpm

sudo rpm -ivh mysql57-community-release-el7.rpm

sudo yum install -y mysql-server

sudo yum install mysql-devel

root 账户下 systemctl start mysqld service mysqld start

修改密码 方法1

cat /var/log/mysqld.log | grep password

mysql -u root -p

ALTER USER ‘root’@’localhost’ IDENTIFIED BY ‘Xs11111111!’;

修改密码 方法2

在root账户下执行下拉命令

vi /etc/my.cnf

找到[mysqld]，在下面添加一行skip-grant-tables

service mysqld restart

mysql -uroot -p

直接按回车，这时不需要输入root密码

flush privileges;

ALTER USER ‘root’@’localhost’ IDENTIFIED BY ‘Xs11111111!’;

vi /etc/my.cnf 删除 skip-grant-tables

service mysqld restart

修改密码 方法3

mysql -uroot -p

use mysql;

update user set password=password(“Xs11111111!”) where user=’root’;

flush privileges;

exit;

8 安装nginx

在根目录下面执行（~）

wget http://nginx.org/download/nginx-1.6.2.tar.gz

tar zxvf nginx-1.6.2.tar.gz

cd nginx-1.6.2

./configure –prefix=/usr/local/nginx –with-http_stub_status_module –with-http_ssl_module

在安装的过程中看缺少什么包，用下面类似的命令安装缺少的包（sudo yum -y install pcre pcre-devel 然后在重新 ./configure）

make

sudo make install

/usr/local/nginx/sbin/nginx -v

9 配置nginx，需要在root账户下

配置 /usr/local/nginx/conf/nginx.conf

检查配置是否正确 /usr/local/nginx/sbin/nginx -t

启动 /usr/local/nginx/sbin/nginx

访问域名或者ip

/usr/local/nginx/sbin/nginx -s reload # 重新载入配置文件

/usr/local/nginx/sbin/nginx -s reopen # 重启 Nginx

/usr/local/nginx/sbin/nginx -s stop # 停止 Nginx

ps -ef | grep nginx 查看进程

kill -9 123123（第一个数字）

tail -f /usr/local/nginx/logs/error.log 查看nginx的错误日志

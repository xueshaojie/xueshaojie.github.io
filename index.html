git checkout – app/models/product.rb （撤销已经改变的app/models/product.rb文件）

新建一个分支开发,提交到远程仓库之前需要先合并

git checkout -b ch01 （新建ch01分支并且切换到ch01，接下来进行开发。）
git status （查看所修改的文件）
git diff （查看所修改文件内的具体内容）
git add .
git commit -m “” （引号里面写入你改代码的总结。例如，增加了index控制器。）
git checkout master （切换到另外master分支）
git merge ch01 合并 （把ch01已经提交的内容，合并到master分支上面。）
git pull –rebase （本地有commit之后，不要直接git pull，而是需要使用git pull –rebase。）
git push （推送到远程）

Ruby学习笔记（三）

块是什么

块就是在调用方法是，能与参数一起传递的多个处理的集合。接收块的方法会执行必要次数的块。块的执行次数由方法本身决定。

块的调用方法一般采用以下形式：

块的开头是块变量。块变量就是在执行块的时候，从方法传进来的参数。不同方法的块变量个数也不相同。

块的使用方法

在接收块的方法中，实现了循环处理的方法称为迭代器。each方法是一个典型的迭代器。

散列会将[key, value]的组合作为数组来提取元素。

在接收块变量时，多重赋值规则也是同样适用的。

File对象被用于读写文件的内容。使用File对象可将文件数据从头到尾读取出来。

根据文件内容的不同，我们需要考是以字符为单位，还是以行为单位来做读取处理。除了each_line方法外，File对象中还有以字符为单位来循环读取数据的each_char方法，以及以字节为单位进行循环读取的each_byte方法等等。

确保后处理被执行

文件使用块时，有方法执行关闭操作，而我们只需将必要的处理记述在块中即可。这样一来可以减少程序的代码量，二来可以防止忘记关闭文件等错误发生。

替换部分算法

Array类的sort方法是对数组内元素进行排序的方法，对数组元素进行排序，可以采取多种方法。

按数字的大小顺序，按字母顺序，按字符串的长度顺序，按数组元素的合计值的大小顺序。

Array#sort方法没有指定块时，会使用<=>运算符对各个元素进行比较，并根据比较后的结果进行排序。<=>运算符的返回值为-1、0、1中的一个。

块的最后一个表达式不是指块的最后一行表达式，而是指在块中最后执行的表达式。

定义带块的方法

在接受块的方法中执行块时，可以使用yield关键字。而Ruby还能把块当作对象处理。把块当作对象处理后，就可以在接受块的方法之外的其他地方执行块，或者把块交给其他方法执行。

在块中使用break，程序会马上返回到调用块的地方。如果在块中使用next，程序就会中断当前处理，并继续执行下面的处理。使用next后，执行块的yield会返回，如果next没有指定任何参数则返回nil，而如果想next 0 这样指定参数，那么该参数值就是返回值。

Proc对象是能让块作为对象在程序中使用的类。定义Proc对象的典型的方法是，调用Proc.new方法这个带块的方法。在调用Proc对象的call方法之前，块中定义的程序不会被执行。

把块从一个方法传给另一个方法时，首先会通过变量将块作为Proc对象接受，然后再传给另一个方法。在方法定义时，如果末尾的参数使用“&参数名”的形式，Ruby就会自动把调用方法时传进来的块封装为Proc对象。

我们也能将Proc对象作为块传给其他方法处理。这时，我们只需在调用方法时，用”&Proc对象”的形式定义参数就可以了。

局部变量和块变量

块内部的命名空间与块外部是共享的。在块外部定义的局部变量，在块中也可以继续使用。而被作为块变量使用的变量，即使与块外部的变量同名，Ruby也会认为它们是两个不同的变量。在块内部定义的变量不能被外部访问。

在块内部给局部变量赋值的时候，要时刻注意它与块外部的同名变量的关系。大家一定要小心Ruby中的这个陷阱。

块变量是只能在块内部使用的变量（块局部变量），它不能覆盖外部的局部变量，但Ruby为我们提供了定义块变量以外的块局部变量的语法。通过在块变量后使用;加以区分的方式，来定义块局部变量。

Ruby学习笔记（二）

类和实例

类表示对象的种类。Ruby中的对象都一定属于某个类。

类就像是对象的雏形和设计图。

我们在生成新的对象时，一般会用到各个类的new方法。

class关键字在定义类时使用。

当想知道某个对象属于哪个类是，我们可以使用class方法。

当判断某个对象是否属于某个类时，我们可以使用Instance_of? 方法。

根据类的继承关系反向追查对象是否属于某个类时，则可以使用is_a? 方法。

initialize方法

使用new方法生成新的对象时，intiialize方法会被调用，同时new方法的参数会原封不动地传给initialize方法。因此初始化对象时需要的程序都写在这个方法中。

存取器的定义

定义 意义
attr_reader :name 只读（定义name方法）
attr_writer :name 只写（定义name= 方法）
attr_accessor :name 读写（定义以上两个方法）

Ruby中一般把设定实例变量的方法称为writer，读取实例变量的方法称为reader方法，这两个方法合称为accessor。有时把reader称为getter，writer称为setter，合称为accessor method。

实例方法和实例变量

我们把已@开头的变量称为实例变量。在不同的方法中，程序会把局部变量看作是不同的变量来对待。而只要在同一个实例中，程序就可以超越方法定义，任意引用、修改实例变量的值。另外，引用未初始化的实例变量是的返回值为nil。

不同实例的实例变量值可以不同。只要实例存在，实例变量的值就不会消失，并且可以被任意使用。而局部变量则是在调用方法时被创建，而且只能在该方法内使用。

类方法和类变量

方法的接受这就是类本身（类对象）的方法称为类方法。

class << 类名 ~ end这种写法的类定义成为单例类定义，单例类定义中定义成的方法称为单例方法。

以@@开头的变量称为类变量。类变量是该类所有实例的共享变量，这一点与常量类似，不同的是我们可以多次修改类变量的值。从类的外部访问类变量是需要存取器。（自己定义）

继承

定义时继承时，在使用class关键字指定类名的同时指定父类名。

alias与undef

alias的使用场景：1，为方法设置别名。2，在重定义已经存在的方法时，为了能用别名调用原来的方法。

undef用于删除已有方法的定义。

单例类

单例类定义被用于定义对象的专属实例方法。

Ruby中所有的类都是Class的实例，对Class类添加实例方法，就等于给所有的类都添加了该类方法。因此，只希望对每个实例添加方法时，就需要利用单例方法。

限制方法的调用

public 以实例方法的形式向外部公开该方法。

private 在指定接收者的情况下不能调用该方法。（无法从实例的外部访问。）

protected 在同一个类中可将该方法作为实例方法调用。

没有指定访问级别的方法默认为public，但initialize方法是个例外，它通常会被定义为private。

模块

模块与类的区别：模块不能拥有实例，模块不能被继承。

模块的使用方法

通过在模块内定义名称，就可以解决命名冲突的问题。

通过include可以把模块内的方法名、常量名合并到当前的命名空间。

Mix-in就是讲模块混合到类中。在定义类的时使用include，模块里的方法，常量就都能被类使用。

如果希望把模块作为模块函数公开给外部使用，就需要用到module_function方法。module_function的参数是表示方法名的符号。

如果希望把模块做完模块函数公开给外部使用，就需要用到module_function方法。module_function的参数是表示方法名的符号。

Mix-in

如果想知道类是否包含每个模块时，可以使用include？方法。被包含的模块的作用就类似于虚拟的父类。

查找方法

方法查找自内而外，自上而下。模块是一个特殊的类，不能实例化。

同继承关系一样，原类中已经定义了同名的方法时，优先使用该方法。

在同一个类中包含多个模块时，优先使用最后一个包含的模块。

嵌套include时，查找顺序也是线性的。

相同的模块把包含两次以上时，第2次以后的会省略。

类与Mix-in

module：声明一个module。include：把一个module的方法注入为实例方法。include可以帮助我们突破继承的限制，通过模块扩展类的功能。extend：把一个module的方法注入为类方法。extend可以帮助我们跨过类，直接通过模块扩展实例的功能。

include把模块注入当前类的继承链后面。

prepend把模块注入当前类的继承链前面。

最后自己做错，而后寻找答案。自己今天下午在自己的电脑上面，已经做成功了。可是当自己再次做错之后，自己发现按照自己原以为的答案并不能解决问题，通过自己反复试错之后，最终形成了答案。

这次整理发现了很多问题。一个正常的现象是：自己也不知道自己就把这个问题解决掉了。有可能，这里搜索一个命令，那里搜索一个答案，最终解决了自己所遇到的问题，可是自己最终还是不知道真正起作用的那些命令。有可能自己心里以及知道了那些命令真起作用了，可是自己并不能再进行尝试，很可能自己以为起作用的命令并非是真正起作用的命令。

当自己整理自己的错误，并且形成文章的时候，自己必须要保证，自己所说的解决方法是真正有效的，进一步让自己知道对于这个问题，那些命令是真正起作用的。

学习全栈的心得

有些事情，自己经历过后，自己才能有切身的体会。

在自己开始学习全栈的时候，老师就给出了建议：记录学习过程当中的错误。在刚开始的时候，自己还挺重视这条建议。不过，这里的重视只是在心里上面的重视而已，并没有给予实际的行动。

最近这个星期开始学习计算机的过程中，自己也会遇到一些错误，不过自己总是认为没有必要记录，大部分错误来自自己拼写的错误。直到今天遇到一个错误，花费了自己3个小时，才最终把这个错误解决。实际上，自己以前也遇到过同样的问题，自己只知道自己输入了四行命令之后，最终把那个问题解决掉了。可是由于自己并没有记录的习惯，自己不能完全回忆起那四行命令到底是什么。自己在Google上面搜索关键字，并没有找到那四行命令。真实的情况是，那四行命令进行了记录，不过是记录在一张纸上，最近回到家中，并没有带那张纸。

正确的步骤是：gem ‘devise’

bundle install

rails g devise:install

rails g devise user

rake db:migrate

完成之后 重新开 rails s

自己并没有执行rails s命令，随后自己执行了rails d devise:install 再执行rails d

devise user出现了下列的错误。

解决方法是：注释掉 app/models/user.rb 当中第4和第5行。注释的方法是：第4行和第五行前面#。或者是将要注释掉的内容删掉。

注释掉config/routes.rb当中第2行。

删除掉db/migrate/一串数字_devise_create_users.rb。

终端机执行：

rails d devise user

rails d devise:install

重新执行

rails g devise:install

rails g devise user

rake db:migrate

发现上述错误，执行 rake db:drop:all，再执行rake db:migrate

终于把这个问题解决掉了，自己今天下午花了三个小时的时间，自己的电脑上解决了这个问题，可是当自己写错误记录的时候，要遇到了很多问题。

第一个问题是，自己图片瞎改，由于自己不会使用mac当中的预览app，一不小心给截图上面乱画了东西，自己找了半天解决方法，最终没有找到解决方法。

最后自己做错，而后寻找答案。自己今天下午在自己的电脑上面，已经做成功了。可是当自己再次做错之后，自己发现按照自己原以为的答案并不能解决问题，通过自己反复试错之后，最终形成了答案。

这次整理发现了很多问题。一个正常的现象是：自己也不知道自己就把这个问题解决掉了。有可能，这里搜索一个命令，那里搜索一个答案，最终解决了自己所遇到的问题，可是自己最终还是不知道真正起作用的那些命令。有可能自己心里以及知道了那些命令真起作用了，可是自己并不能再进行尝试，很可能自己以为起作用的命令并非是真正起作用的命令。

当自己整理自己的错误，并且形成文章的时候，自己必须要保证，自己所说的解决方法是真正有效的，进一步让自己知道对于这个问题，那些命令是真正起作用的。

留言

股票价格的变动是否可以准确预测？

我的理解是，股票价格的变动预测准确和预测不准确，实际上并不是最重要的事情。我认为更有价值的问题是，股票价值的变动是否可以准确预测？价格本身只是衡量价值的一种物品而已。

预测股票价格的变动，只要我们预测，不管我们是否可以预测准确，我们都会花费大量的时间和精力持续关注股票的价格，这样我们就会得不偿失。

我们应该思考怎么预测股票价值的变动？只要我们预测准确了股票的价值，那么股票的价格我们自然而得知。价值相对持续周期比较长，而价格的变动是在分秒变化的。预测价值，并不需要我们持续关注，我们可以把自己的注意力放在自己的成长之上。

股票的价格只是在股票的价值附近波动而已。
